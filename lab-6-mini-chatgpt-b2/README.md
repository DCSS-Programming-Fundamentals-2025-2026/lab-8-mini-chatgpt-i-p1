# MiniChatGPT — Модуль навчання та обробки даних (B2)

Цей репозиторій — результат інтеграції двох підсистем (команди A3 та A9) у межах лабораторної роботи №8.
Основна задача — об’єднати підготовку батчів (batching) з процесом навчання мовних моделей.

## Структура проєкту

Проєкт оформлений як одне solution-рішення і складається з трьох частин:

src/Lib.Batching (Team A3)
Модуль відповідає за підготовку даних — перетворює потік токенів у випадкові батчі.
Використовуються інтерфейси ITokenStream та IBatchProvider.

src/Lib.Training (Team A9 + доробки)
Містить логіку навчання моделей.
Основні елементи:
TrainingLoop
інтерфейси моделей (NGram, NeuralNetwork)
tests/IntegrationTests
Інтеграційні тести (9 штук), які перевіряють, як модулі працюють разом.
Особливості інтеграції

Під час об’єднання модулів довелось внести кілька змін, щоб усе стабільно працювало:

**Тип токенів**
У модулі навчання string було замінено на int.
Це зроблено для сумісності з batching-модулем, який працює з числовими ID токенів.
Одна версія платформи
Усі проєкти переведені на .NET 9.0, щоб уникнути проблем із залежностями.
Різні підходи до навчання
Реалізовано окрему логіку залежно від типу моделі:
для нейромережі (TinyNN) — навчання через батчі
для NGram — робота напряму з повним потоком токенів
Обробка помилок
Основний цикл навчання обгорнутий у try-catch, щоб не падати при некоректних даних (порожній текст, пошкоджені дані і т.д.)
Тестування

**Є 9 інтеграційних тестів, розбитих на категорії:**

Functional
Перевірка повного процесу навчання для різних типів моделей
Edge cases
нульова кількість епох
дуже короткі тексти
порожні вхідні дані
Stability
перевірка відтворюваності (через Seed)
коректність передачі параметрів (LearningRate, BatchSize)
