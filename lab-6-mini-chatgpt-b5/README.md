# 📄 Specific Documentation: Оновлення TinyTransformer (А8)

Цей документ описує конкретні архітектурні зміни та оновлення, які були внесені до підсистеми **TinyTransformer** (команда А8) під час переходу з першого етапу розробки.

Ці зміни були необхідні для забезпечення повної сумісності з об'єднаною екосистемою проєкту (зокрема з `Lib.MathCore`) та для підтримки життєвого циклу моделі.

---

### 1. Перехід на використання типу `float` замість `double`

* **Суть зміни:** У всіх математичних обчисленнях, масивах конфігурацій, векторах (Embeddings) та матрицях ваг відбулася повна міграція з типу даних `double` на `float`.
* **Причина:** Це рішення було прийнято для уніфікації кодової бази. Оскільки загальне математичне ядро `Lib.MathCore` побудоване на використанні `ReadOnlySpan<float>`, перехід на `float` гарантує безшовну інтеграцію між підсистемами та уникає зайвих конвертацій типів (cast) під час виконання. Крім того, це вдвічі зменшує обсяг споживаної оперативної пам'яті матрицями моделі.

### 2. Додавання логіки збереження чекпоінтів

* **Суть зміни:** Модель отримала здатність зберігати свій поточний стан. Для цього до головного класу моделі було додано метод `model.GetPayloadForCheckpoint()`.
* **Як це працює:** Метод збирає всі необхідні параметри конфігурації (VocabSize, EmbeddingSize тощо) та поточні значення матриць ваг (включаючи матриці Q, K, V механізму уваги та лінійних шарів). Усі ці дані пакуються в єдиний об'єкт Payload, який ідеально підходить для подальшої серіалізації у формат JSON та запису на диск.

### 3. Відновлення моделі через `TinyTransformerModelFactory`

* **Суть зміни:** Для забезпечення повного циклу "збереження-відновлення" (Round-Trip) функціонал фабрики створення моделей було розширено. До класу `TinyTransformerModelFactory` додано новий метод `FromPayload(JsonElement payload)`.
* **Як це працює:** Метод приймає сирий `JsonElement`, який система читає з файлу чекпоінту. Фабрика розбирає цей JSON, десеріалізує налаштування та відновлює масиви ваг. На виході метод повертає повністю ініціалізований та готовий до роботи екземпляр `TinyTransformerModel`, стан якого абсолютно ідентичний стану моделі на момент її збереження.
